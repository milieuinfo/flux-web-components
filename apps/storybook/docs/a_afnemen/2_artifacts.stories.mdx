## Overzicht

Alle web-componenten van Departement Omgeving zijn beschikbaar via npm artifacts onder de npm scope `@domg-wc`;
__common__ bevat technische code, __styles__ alle styling en layouts die op het document gezet worden,
__components__ en __map__ bevatten de web-componenten. De web-componenten zijn ook beschikbaar via de CDN, zie
['fat-js: domg-wc'](#fat-js-domg-wc).

> **⚠ Waarschuwing**: er is een elements / components concept gedefinieerd, dit is een streefdoel, de huidige code
volgt dit concept in grote lijnen maar niet volledig. Afwijkingen komen voor!


## @domg-wc/common

__common__ is een technische bibliotheek met basis code en utilities


## @domg-wc/components/block

Historisch werd een __component__ geïmplementeerd als een `BaseElement`: een eigen basis klasse beschikbaar in
@domg-wc/common via de methode `BaseElementOfType`. Nieuwere componenten worden geïmplementeerd m.b.v.
[LitElement](https://lit.dev/docs/api/LitElement/); het doel op (lange) termijn is van alles (ook van de compliance en map) LitElementen te maken.

Een __component__ heeft technisch volgende eigenschappen:

- extends van BaseElement of van LitElement
- dus steeds te gebruiken als custom tag; bvb. `<vl-breadcrumb></vl-breadcrumb>`
- heeft __steeds__ een shadow DOM
- de styling komt mee met de component (wordt geïnjecteerd in de shadow DOM)
- element styling wordt waar nodig voorzien in de shadow DOM


## @domg-wc/components/form

Bevat form-gerelateerde web-componenten.


## @domg-wc/components/compliance

Onder __components/compliance__ zitten web-componenten specifiek voor Departement Omgeving. Het zijn geen bouwstenen
zoals de andere __componenten__. Volgende componenten zitten hieronder: __vl-header__, __vl-footer__,
__vl-accessibility__, __vl-cookie-consent__, __vl-cookie-consent-opt-in__, __vl-cookie-statement__ en __vl-privacy__.


## @domg-wc/map

De __map__ is de grootste en uitgebreidste component. Het is een kaart component die achterliggend
[OpenLayers v8.2.0](https://openlayers.org) gebruikt. De functionaliteit zit in sub-componenten
die een plugin opzet volgen.


## fat-js: domg-wc

Voor legacy toepassingen is er een 'fat-js' variant van de web-componenten beschikbaar. Via de CDN van Departement
Omgeving kan je daar geversioneerd naar refereren conform: `https://cdn.omgeving.vlaanderen.be/domg/domg-wc/1.27.0/domg-wc-1.27.0.min.js`.
Bij de import worden alle web-componenten (components en map) geregistreerd. Zie
[CDN Bundel - fat-js](/docs/recepten-cdn-bundel-fat-js--documentatie) onder `Cookbook` voor meer informatie.
